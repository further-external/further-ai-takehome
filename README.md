## **AI Engineer — Technical Take-Home Assignment**

## **Overview**

Build an AI agent in one of the domains below. The agent should feel meaningfully useful—not just a wrapper around a prompt. We’re evaluating how you think about architecture, tooling, and product design, not domain expertise, so pick whichever domain excites you most.

**Choose one:**

|                                                                                                                                                            |                                                                                                                                                                              |                                                                                                                                                                               |
| :--------------------------------------------------------------------------------------------------------------------------------------------------------: | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------: | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------: |
| **🚗 Car Purchasing Agent** Help a user buy a car. Think budget analysis, vehicle comparison, financing options, depreciation modeling, or purchase timing. | **✈️ Travel Planning Agent** Help a user plan a trip. Think itinerary optimization, budget allocation, destination comparison, travel logistics, or packing recommendations. | **🍽️ Meal Planning Agent** Help a user plan their meals for the week. Think dietary constraint satisfaction, grocery budgeting, nutrition optimization, or recipe scheduling. |

*These are starting points—feel free to narrow the scope or take a creative angle within your chosen domain.*

## **What We’re Evaluating**

  * **Agent architecture:** How you structure the agent, its tools, state management, and decision-making flow. We care about thoughtful design over complexity.
  * **Computational tooling:** We value tools that compute answers—not just retrieve content. Show us your logic. If a tool optimizes a budget, scores options by preference fit, or models trade-offs, that’s what we want to see.
  * **Code quality:** Clean, readable, well-organized code with clear separation of concerns. We’ll be reading your code as closely as we run it.
  * **Product thinking:** Does the agent actually help someone? Does it feel thoughtful? We build production AI systems for enterprise clients—we’re looking for engineers who think beyond the demo.

## **What’s Flexible**

  * **UI is optional** — feel free to vibe code it, use the default LangGraph Studio UI, a CLI, or skip it entirely.
  * **Doesn’t need to be conversational** — a workflow-based or task-based agent is fine.
  * **Tech stack is your choice** — use whatever you’re most productive with. However, we ask that you **please use the LangGraph dev server** so that we can effectively review your code.
  * **Model provider:** We’ll ask that you create an account at ai.google.dev and obtain a free API key. You’re also welcome to use your own keys for any provider.

## **Submission Instructions**

1.  **We’ll add your GitHub username** to the further-ai-takehome repo with Read access.
2.  **Clone the repo locally**, then push to a new private repository on your own GitHub account.
3.  **Build your solution** in your private repo.
4.  **When ready**, add us as collaborators to your private repo so we can review.

Keep in mind that we will be using LangGraph Studio to evaluate what you build.

## **Time Expectation**

Don’t over-engineer it—we’d rather see a focused, well-executed slice than a sprawling half-finished project. If you run out of time, document what you would have done next and why. We want you to have the opportunity to **show where you can really shine**. Feel free to call that out so we know where to focus.

The use of AI-assisted coding tools (Cursor, Claude Code, Copilot, etc.) is encouraged. We use them at Further every day and are always looking for engineers who know how to leverage them effectively.

## **Questions?**

If anything is unclear, reach out. We’re happy to clarify scope or answer questions. Good luck\!
